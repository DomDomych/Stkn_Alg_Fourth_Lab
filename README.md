# Лабораторная Работа №4. Cокращатель Ссылок  

В программе реализован сокращатель ссылок, который позволяет сохранять длинные ссылки, выдавать для них короткие коды, находить исходную ссылку по короткому коду, проверять существование короткого кода и выводить все сохранённые пары.  

## Использованная Структура Данных  

В качестве основной структуры данных используется AVL-дерево. Это самобалансирующееся бинарное дерево поиска. Оно выбрано потому, что обеспечивает логарифмическую сложность поиска, вставки и удаления, а также позволяет выводить данные в отсортированном порядке по ключу. Для данной задачи ключом является короткий код, а значением — исходная длинная ссылка.

В проекте используется два AVL-дерева. Первое дерево хранит соответствия короткий код — длинная ссылка. Оно нужно для основной операции сокращателя: по короткому коду найти исходную ссылку. Второе дерево хранит обратные соответствия: длинная ссылка — короткий код. Оно нужно для того, чтобы при повторном добавлении уже существующей длинной ссылки не создавать новый короткий код, а возвращать старый.  

## Генерация Кодов  

Короткие коды генерируются автоматически на основе счётчика. При добавлении первой ссылки создаётся код a, при добавлении второй — b, затем c и так далее. Для генерации используется алфавит из строчных букв, заглавных букв и цифр. Такой подход позволяет получать компактные коды и постепенно увеличивать их длину при росте количества ссылок.  

## Реализованные Требования  

Обязательная часть задания реализована полностью. Программа умеет добавлять новую ссылку, получать длинную ссылку по короткому коду, проверять существование короткого кода и выводить все сокращённые ссылки.  

Также реализованы дополнительные пункты вариативной части. Запрещено создание двух одинаковых коротких кодов, так как коды генерируются последовательно на основе счётчика. Если длинная ссылка уже существует, программа возвращает старый короткий код, а не создаёт новый. Реализовано удаление ссылки по короткому коду. При удалении запись удаляется из обоих деревьев, чтобы прямое и обратное соответствия оставались согласованными.  

Для каждой короткой ссылки подсчитывается количество переходов. Счётчик переходов хранится в вершине дерева. Когда пользователь получает исходную ссылку по короткому коду, счётчик этой ссылки увеличивается. На основе этих счётчиков реализован вывод самой популярной короткой ссылки.  

Реализован поиск всех ссылок, содержащих заданное слово. Для этого выполняется обход дерева, а каждая длинная ссылка проверяется с помощью алгоритма Кнута — Морриса — Пратта. Этот алгоритм используется для поиска подстроки в строке и позволяет не начинать проверку заново после каждого несовпадения. В рамках данной задачи он проверяет, содержится ли заданное слово внутри длинной ссылки.  

Вывод всех пар выполняется в отсортированном порядке по короткому коду. Это достигается за счёт симметричного обхода AVL-дерева. Сначала обходится левое поддерево, затем текущая вершина, затем правое поддерево.  

Реализовано ограничение максимальной длины короткого кода. Если сгенерированный код превышает допустимую длину, новая ссылка не добавляется. Минимальная длина кода фактически равна одному символу, так как генератор не создаёт пустые коды.  

Также реализовано изменение длинной ссылки для существующего короткого кода. При изменении программа сначала проверяет, существует ли такой короткий код. Затем удаляет старое обратное соответствие из дерева длинная ссылка — короткий код, обновляет прямое соответствие и добавляет новое обратное соответствие. Это необходимо, чтобы два дерева оставались согласованными.  

Реализован обратный поиск: по длинной ссылке можно найти короткий код. Для этого используется второе AVL-дерево, где длинная ссылка является ключом, а короткий код — значением.  

### Структура Проекта и Его Сборка  

Проект разделён на несколько частей. Модуль дерева содержит реализацию AVL-дерева, его узлов, вставки, поиска, удаления, балансировки, поворотов, обхода и дополнительных операций. Модуль сокращателя ссылок содержит бизнес-логику: генерацию кодов, добавление ссылок, получение исходной ссылки, обратный поиск, изменение и удаление ссылок. Файл main.cpp демонстрирует работу всех основных возможностей программы.  

Для сборки проекта используется CMake. В демонстрационном файле показано добавление ссылок, повторное добавление существующей ссылки, получение исходной ссылки по короткому коду, проверка существования кода, подсчёт переходов, вывод самой популярной ссылки, поиск ссылок по слову, обратный поиск, изменение ссылки, удаление ссылки и финальный вывод всех сохранённых записей.  

### Итог  

В результате была реализована полноценная учебная версия сокращателя ссылок на основе AVL-дерева. Программа закрывает обязательную часть задания и расширенную вариативную часть. Основная структура данных реализована самостоятельно, включая балансировку, поиск, вставку, удаление и обход дерева.
